# Space Invaders — TouchGFX / STM32F429I-DISCO

A Space Invaders remaster built with **TouchGFX** on the **STM32F429I-DISCOVERY** board (custom Rev D01/E01 TBS), featuring a redesigned UI, tiered weapon/explosion effects, and a session menu.

## Hardware

| Item | Value |
|---|---|
| Board | STM32F429I-DISCO (Rev D01/E01 template board support, TBS) |
| MCU | STM32F429ZIT6 (LQFP144) |
| Display | 320 × 240, 16 bpp (RGB565), LTDC + DMA2D |
| RAM | External SDRAM via FMC |
| RTOS | FreeRTOS (CMSIS-V2 wrapper) |

Performance-testing GPIO pins are broken out for logic-analyzer probing: `VSYNC_FREQ` (PE2), `RENDER_TIME` (PE3), `FRAME_RATE` (PE4), `MCU_ACTIVE` (PE5).

## Required tools & versions

The project was developed/generated with the following toolchain. Using materially older versions of STM32CubeMX / TouchGFX Designer may fail to regenerate the project correctly.

| Tool | Version | Purpose |
|---|---|---|
| [STM32CubeMX](https://www.st.com/en/development-tools/stm32cubemx.html) | 6.14.0 | Pin/clock/middleware configuration (`SpaceInvaders.ioc`), regenerates `Drivers/` and `Middlewares/` |
| STM32Cube Firmware Package | FW_F4 V1.28.1 | HAL/LL drivers + FreeRTOS for STM32F4 |
| [TouchGFX Designer](https://www.touchgfx.com/) (X-CUBE-TOUCHGFX) | 4.25.0 | UI design, screens, widgets (`TouchGFX/SpaceInvaders.touchgfx`), code generation |
| [STM32CubeIDE](https://www.st.com/en/development-tools/stm32cubeide.html) | 1.15.x or newer (Eclipse/GCC based) | Default IDE — build & debug (`STM32CubeIDE/`) |
| [STM32CubeProgrammer](https://www.st.com/en/development-tools/stm32cubeprog.html) | Latest | Flashing the board (also used by TouchGFX Designer's "Run" button) |
| Git | Any recent version | Version control |
| PowerShell 5.1+ / 7+ (Windows) | — | Only needed to run the asset-generation helper scripts in `tools/` (uses `System.Drawing`) |

Alternative IDEs are also supported by the `.ioc` (select in STM32CubeMX → Project Manager → Toolchain/IDE):

- **IAR EWARM** — 8.50.9 or newer (`EWARM/`)
- **Keil MDK-ARM** (`MDK-ARM/`)
- **Plain GCC / Makefile** (`gcc/`)

## Repository layout

```
Core/                   Application C sources (main.c, HAL config, FreeRTOS glue)
TouchGFX/
  SpaceInvaders.touchgfx  TouchGFX Designer project (screens, widgets, interactions)
  gui/                    Hand-written view/presenter code (game logic)
  generated/              Code generated by TouchGFX Designer (bitmaps, base classes) — do not hand-edit generated bases
  assets/images/          Source PNG assets for sprites/effects
STM32CubeIDE/           STM32CubeIDE project (default IDE)
EWARM/                  IAR EWARM project
MDK-ARM/                Keil MDK-ARM project
gcc/                    Plain Makefile/GCC build
Drivers/, Middlewares/  Vendor HAL/CMSIS/FreeRTOS/TouchGFX libs — gitignored, regenerated from SpaceInvaders.ioc
tools/                  PowerShell scripts used to generate tiered bullet/explosion/hit-spark sprite assets
SpaceInvaders.ioc       STM32CubeMX project (MCU pinout, clocks, middleware)
```

`Drivers/` and `Middlewares/` are intentionally excluded from git (see `.gitignore`) — they are regenerated automatically from `SpaceInvaders.ioc` by STM32CubeMX/TouchGFX Designer.

## Setup & installation

1. Install **STM32CubeMX** and **STM32CubeIDE** (or IAR EWARM / Keil MDK-ARM if you prefer a different IDE).
2. Install **TouchGFX Designer** — either standalone, or via the X-CUBE-TOUCHGFX pack inside STM32CubeMX (Help → Manage embedded software packages).
3. Install **STM32CubeProgrammer** (required for flashing the board, from either STM32CubeIDE or TouchGFX Designer).
4. Clone this repository:

   ```bash
   git clone https://github.com/leviethoa19/Space-Invaders.git
   cd Space-Invaders
   ```

5. Generate the vendor code (only needed the first time, or after changing `SpaceInvaders.ioc`):
   - Open `SpaceInvaders.ioc` in STM32CubeMX and click **GENERATE CODE**. This creates `Drivers/` and `Middlewares/`.

## Building & flashing

**Using STM32CubeIDE (default):**

1. Open STM32CubeIDE and `File → Import → Existing Projects into Workspace`, pointing at `STM32CubeIDE/`.
2. Build with `Project → Build Project` (or Debug/Release build configuration).
3. Connect the STM32F429I-DISCO board via its USB ST-LINK port and click **Run**/**Debug** to flash.

**Using TouchGFX Designer:**

1. Open `TouchGFX/SpaceInvaders.touchgfx`.
2. Edit screens/widgets as needed, then generate code.
3. Use the **Build** / **Run** button in TouchGFX Designer to build with GCC and flash directly via STM32CubeProgrammer.

**Using STM32CubeProgrammer manually:**

- Build a `.bin`/`.hex` with any of the supported IDEs, then flash it via STM32CubeProgrammer (SWD, ST-LINK).

## Asset generation scripts (`tools/`)

The `tools/*.ps1` scripts (Windows PowerShell) regenerate the tiered bullet/explosion/hit-spark sprites (`image_*.cpp` under `TouchGFX/generated/images/src/`) from source artwork. Preview renders are written to `tools/preview/`. These are optional developer utilities — you only need them if you're modifying the sprite effects.
