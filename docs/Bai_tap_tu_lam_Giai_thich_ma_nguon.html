BÀI 3.1 – Giải thích hoạt động chương trình (cuối phần 3.1.3)

Project: HelloTouchGFX (Bai313Lab3)

1. Mục tiêu

Giới thiệu TouchGFX trên STM32F429, kiến trúc MVP, FreeRTOS và Message Queue truyền sự kiện nút bấm PA0 sang luồng giao diện.

2. Hai task FreeRTOS

TaskNhiệm vụ
TouchGFX_TaskChạy TouchGFX: vẽ widget, animation, xử lý cảm ứng
defaultTaskĐọc nút PA0, gửi ký tự 'X' vào myQueue01 khi nhấn

3. Khởi động (main.c)

  1. HAL_Init, cấu hình clock 180 MHz
  2. LTDC, DMA2D, SDRAM (framebuffer LCD)
  3. MX_TouchGFX_Init()
  4. osMessageQueueNew tạo myQueue01Handle
  5. Tạo task và osKernelStart()

4. StartDefaultTask – đọc nút PA0

for (;;)
{
    if (HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_0) == GPIO_PIN_SET)
    {
        uint8_t data = 'X';
        osMessageQueuePut(myQueue01Handle, &data, 0, 10);
    }
    osDelay(1);
}

Task nền chỉ ghi queue, không vẽ LCD trực tiếp.

5. Screen1View::tickEvent()

  • tickCount += 2, modulo 240
  • Nếu queue có dữ liệu và nhận 'X': di chuyển circle1
  • Quỹ đạo: y = sin(x) + sin(2x) + sin(3x) + 1
  • circle1.moveTo(...) + invalidate()

6. Screen1View::buttonClicked()

Nút cảm ứng trên LCD → HAL_GPIO_TogglePin(GPIOG, GPIO_PIN_13) (LED đỏ board).

7. Luồng dữ liệu

PA0 → defaultTask → myQueue01 → tickEvent() → circle1 animation → LCD
Nút cảm ứng → buttonClicked() → LED PG13

8. Kết luận 3.1

Minh họa TouchGFX + FreeRTOS + Message Queue: tách task đọc GPIO và task GUI, animation widget kết hợp sự kiện phần cứng.

BÀI 3.2 – Giải thích hoạt động chương trình

Project: StopWatch (Bai32Lab3)

1. Mục tiêu

Ứng dụng đồng hồ bấm giờ: kim quay theo thời gian giữ nút PA0, dùng TextureMapper và osKernelGetTickCount().

2. Queue và task nền

myQueue01Handle = osMessageQueueNew(1, sizeof(uint8_t), ...);

for (;;)
{
    buttonState = HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_0);
    osMessageQueuePut(myQueue01Handle, &buttonState, 0U, 0U);
    osDelay(10);
}

Gửi liên tục trạng thái 0/1 mỗi 10 ms (khác bài 3.1 chỉ gửi khi nhấn).

3. Giao diện

textureMapper1 hiển thị ảnh kim phút analog clock, xoay quanh trục Z.

4. Screen1View::tickEvent()

BiếnÝ nghĩa
stopwatchMsTổng thời gian đo (ms)
lastKernelTickTick kernel lần trước
buttonPressedNút PA0 đang giữ
  1. Đọc hết queue → cập nhật buttonPressed
  2. now = osKernelGetTickCount()
  3. Nếu giữ nút: stopwatchMs += (now - lastKernelTick)
  4. angle = (stopwatchMs % 60000) / 60000 * 2π
  5. textureMapper1.updateZAngle(angle)

5. Luồng hoạt động

PA0 (giữ/nhả) → defaultTask → myQueue01 → tickEvent()
  → cộng dồn stopwatchMs → xoay kim trên LCD

6. So sánh 3.1 và 3.2

3.1 HelloTouchGFX3.2 StopWatch
Queue'X' khi nhấn0/1 liên tục
Chu kỳ task1 ms10 ms
WidgetCircleTextureMapper
LogicAnimation sinĐo thời gian + xoay kim

7. Kết luận 3.2

Kết hợp TextureMapper với đo thời gian FreeRTOS; queue truyền trạng thái nút an toàn giữa hai task.

BÀI TẬP TỰ LÀM – Game đua xe SimpleRacing

Project: SimpleRacing | Joystick ADC PC3/IN13

1. Mục tiêu

  • Đường cuộn từ trên xuống, xe di chuyển ngang
  • Joystick ADC, tăng tốc, chướng ngại random
  • Score, High score (Flash sector 23)

2. File đã sửa

FileNội dung
main.cQueue1, ADC joystick, hiệu chuẩn center
Model.cpp/hppGame state, score, speed, Flash
Screen2View/PresenterGameplay, collision, UI điểm
texts.xmlScore/High + chữ số 0–9

3. Luồng dữ liệu

Joystick VRx → PC3 → ADC1 → defaultTask → Queue1
  → Model::tick() → Screen2Presenter → Screen2View::handleTickEvent()

4. Chi tiết chính

  • main.c: Queue 8×int8_t; dead zone; dir -1/0/+1 mỗi 20 ms
  • Model: score++, gameSpeed++ / 60 tick; saveHighScore Flash 0x081E0000
  • Screen2View: track0–4 animation; xe Y=230; lamb random lane; AABB collision

5. Kết luận

Kế thừa queue FreeRTOS từ bài 3.1/3.2, thay GPIO bằng ADC joystick; mở rộng Model/View cho game đua xe hoàn chỉnh trên MVP TouchGFX.
